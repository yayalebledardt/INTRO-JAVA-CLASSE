# 🚗 Projet Auto+ : Gestion de Concessionnaire

Bienvenue sur le repository **scenar-classes** ! [cite_start]Ce projet Java a été réalisé pour aider le concessionnaire **Auto+** à gérer son parc de 200 véhicules. [cite: 3, 5, 7]

## 📝 Présentation du projet
[cite_start]L'objectif est de passer d'un simple stockage de données dans des variables à une véritable **programmation orientée objet (POO)**. [cite: 10, 39, 55]

## 🛠️ Ce que contient le projet

Le projet est divisé en plusieurs étapes clés (missions) :

* [cite_start]**Mission 1 :** Création d'un premier programme avec des variables simples pour une Peugeot 208. [cite: 10, 15]
* [cite_start]**Mission 3 & 4 :** Création de la classe `Voiture` et apprentissage de l'**instanciation** (création d'objets comme `v1` et `v2`). [cite: 31, 47, 48]
* [cite_start]**Mission 5 :** Utilisation d'un **constructeur** pour créer des voitures plus rapidement en une seule ligne. [cite: 83, 91, 95]
* **Mission 6 :** Association de classes. [cite_start]Chaque `Voiture` possède désormais un objet `Pneu`. [cite: 103, 106]

---

## 💡 Concepts Clés

### 🏗️ La Classe
C'est le **moule** ou le plan de fabrication. [cite_start]Par exemple, la classe `Voiture` définit qu'une voiture a toujours une marque, un modèle et un prix. [cite: 39]

### 🚗 L'Objet
C'est le véhicule réel fabriqué à partir du moule. [cite_start]`v1` est un objet (une Peugeot spécifique), tandis que la classe est le concept général. [cite: 55]

### 🛠️ Le Constructeur
[cite_start]C'est le bloc de code qui permet d'initialiser les caractéristiques d'un objet dès sa création. [cite: 95]
